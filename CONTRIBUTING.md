# Contributing

Thank you for improving GateSignal.

1. Open an issue describing the decision problem, expected behavior, and evidence for the change.
2. Keep preference scores, evidence quality, hard constraints, forecasts, and accountability conceptually separate.
3. Do not add proprietary course, consulting, certification, or company material.
4. Add or update tests for analytical changes.
5. Run `pytest`, `ruff check .`, and `python -m build` before proposing a change.

Examples must be synthetic or clearly licensed for redistribution. References should point to original publications where possible. Changes that imply empirical calibration must include a reproducible validation basis.
